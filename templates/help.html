系统说明

系统概述

本系统是一个基于Flask框架开发的设备管理系统，提供虚拟机和物理机的信息管理、监控检查功能。支持设备信息的增删改查、批量导入/导出，以及通过SSH获取设备系统状态（CPU、内存、磁盘使用率）。系统支持多用户权限管理，包括管理员和普通用户两种角色，并实现了基于部门的权限控制，确保数据安全和访问隔离。

系统版本：V1.4.0

环境依赖

开发语言

  • Python 3.x

核心框架和库

  • Flask (Web应用框架)
  • sqlite3 (数据库，Python标准库)
  • paramiko (SSH客户端，用于获取设备系统信息)
  • openpyxl (Excel文件处理，用于批量导入/导出)
  • pandas (数据处理库，用于Excel文件解析)

安装依赖

pip install flask paramiko openpyxl pandas

配置说明

应用配置

在app.py中，Flask应用配置如下：

功能说明

1. 登录认证

  • 用户登录和退出功能
  • 基于角色的权限管理（管理员/普通用户）
  • 基于部门的权限控制，普通用户只能查看和修改同部门设备
  • 默认用户：管理员(admin/admin123)、普通用户(user/user123)

2. 虚拟机管理

  • 虚拟机信息的增删改查
  • 批量导入/导出虚拟机信息
  • 按条件筛选和搜索（支持产品、部门、组别、虚拟机名称、IP地址、账号、密码、备注）
  • 部门字段支持手动输入或快速选择
  • 权限控制：管理员可查看所有设备密码和执行所有操作；普通用户只能查看和修改同部门设备，密码对其他部门设备隐藏

3. 物理机管理

  • 物理机信息的增删改查
  • 批量导入/导出物理机信息
  • 按条件筛选和搜索（支持部门、设备编码、IP地址、带外IP、账号、密码）
  • 部门字段支持手动输入或快速选择
  • 权限控制：管理员可查看所有设备密码和执行所有操作；普通用户只能查看和修改同部门设备，密码对其他部门设备隐藏

4. 设备状态监控

  • 批量ping检查设备在线状态
  • 获取设备响应延迟（ms）
  • 通过SSH获取CPU、内存、磁盘使用率
  • 资源占用率超过80%时字体标红加粗
  • 所有登录用户均可监控所有设备

5. 账号管理

  • 仅管理员可见
  • 用户列表展示、添加、编辑、删除
  • 部门字段管理，管理员默认"管理"，普通用户默认"无"
  • 部门通过下拉列表选择，从现有设备部门中获取

6. 数据统计

  • 仅管理员可见
  • 展示设备资源使用趋势图表
  • 按季度和年份统计设备数量
  • 支持虚拟机和物理机的数量统计
  • 图表可视化展示，方便直观了解资源使用情况

7. 导航栏功能

  • 顶部导航栏显示系统名称和版本
  • 右上角显示当前用户信息，悬停时显示包含用户角色、部门的下拉菜单
  • 左侧导航栏根据用户角色显示不同的功能菜单
  • 左侧导航栏显示当前页面的选中状态，使用淡蓝色高亮

7. 提示框功能

  • 操作成功或失败时显示Toast提示框
  • 提示框显示位置在页面右上角
  • 支持错误、成功、警告和信息四种类型的提示
  • 提示框2秒后自动消失，不中断用户操作

运行方式

初始化数据库

python init_db.py

启动服务器

python app.py

使用说明

1. 登录系统

  • 访问登录页面：http://127.0.0.1:5000/login
  • 输入用户名和密码
  • 点击登录按钮进入系统

2. 添加设备

  • 在对应设备管理页面点击"添加设备"按钮
  • 填写设备信息，所属部门可手动输入或从下拉列表选择
  • 点击提交保存设备信息

3. 批量导入

  • 下载对应设备的Excel模板
  • 按照模板格式填写设备信息，所属部门可为空
  • 使用"批量导入"功能导入数据
  • 导入成功或失败时会显示提示框

4. 设备状态监控

  • 点击导航栏中的"监控检查"链接
  • 系统自动获取所有设备的状态信息
  • 资源占用率超过80%时会自动标红加粗
  • 所有登录用户均可监控所有设备

5. 账号管理

  • 仅管理员可见账号管理页面
  • 可以添加、编辑、删除用户
  • 设置用户角色（管理员/普通用户）
  • 设置用户所属部门，管理员默认"管理"，普通用户默认"无"
  • 部门从现有设备部门中获取，通过下拉列表选择

6. 查看用户信息

  • 在顶部导航栏右上角点击或悬停用户名
  • 会显示包含用户角色和部门的下拉菜单

7. 刷新数据

  • 在设备管理页面点击"刷新数据"按钮
  • 系统会重新加载设备列表
  • 刷新成功后会显示提示框

8. 权限说明

  • 管理员：可以查看所有设备密码，执行所有操作（添加、查看、修改、删除）
  • 普通用户：只能查看和修改同部门设备，密码对其他部门设备隐藏
  • 普通用户无法删除设备，只有管理员可以删除
  • 普通用户无法访问账号管理页面
  • 监控功能不受部门限制，所有用户均可监控所有设备
  • 操作列按钮根据权限动态显示：普通用户只能看到编辑按钮（同部门设备），管理员可以看到编辑和删除按钮

注意事项

1. SSH连接要求

  • 设备需要启用SSH服务
  • 需要提供正确的用户名和密码
  • 防火墙需要允许SSH连接（默认端口22）

2. 数据安全性

  • 数据库文件建议定期备份
  • 密码以明文形式存储，生产环境建议加密
  • 系统使用SQLite数据库，适合中小型应用场景
  • 基于部门的权限控制确保数据隔离

3. 权限管理

  • 管理员：可以查看所有设备密码，执行所有操作（添加、查看、修改、删除）
  • 普通用户：只能查看和修改同部门设备，密码对其他部门设备隐藏
  • 普通用户无法删除设备，只有管理员可以删除
  • 普通用户无法访问账号管理页面
  • 监控功能不受部门限制，所有用户均可监控所有设备
  • 部门字段管理：管理员默认"管理"，普通用户默认"无"

4. 系统性能

  • 支持管理约1000条设备数据
  • 监控检查功能会同时建立多个SSH连接，可能会消耗较多系统资源
  • 建议在设备数量较多时分批进行监控检查
  • 刷新数据提示采用Toast提示框，2秒后自动消失，不中断用户操作
  • 提示框显示位置在页面右上角，支持错误、成功、警告和信息四种类型
